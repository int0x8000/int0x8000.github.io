Detection-as-Code Pipeline

Sigma rules version-controlled in Git, automatically validated and converted to Graylog Lucene queries via GitHub Actions CI/CD. Every detection change is auditable before it hits production.

SigmaGitHub Actions MITRE ATT&CKOpenSearch GraylogPython

Why Detection-as-Code

Traditional SIEM rule management means clicking through a UI, making changes with no review process, no audit trail, and no way to test a rule before it goes live. If something breaks or produces false positives, there's no clean rollback path and no record of what changed.

Detection-as-Code applies the same engineering discipline to security rules that software teams apply to application code — version control, peer review, automated testing, and consistent deployment. Every rule change goes through Git, gets validated by CI, and produces a deployment artifact before touching production.

Traditional Approach
✗ Rules clicked into SIEM UI manually
✗ No audit trail of changes
✗ No review before going live
✗ Vendor lock-in — rewrite if you switch SIEMs
✗ No way to test before deployment
Detection-as-Code
✓ Rules defined as version-controlled YAML

GitHub Actions Workflow

.github/workflows/sigma-validate.yml
name: Validate and Convert Sigma Rules

on:
  push:
    paths:
      - 'homelab/detection-as-code/rules/**'
  pull_request:
    paths:
      - 'homelab/detection-as-code/rules/**'

jobs:
  sigma-validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v5
      - uses: actions/setup-python@v6
        with:
          python-version: '3.11'

      - name: Install sigma-cli and OpenSearch backend
        run: |
          pip install sigma-cli
          sigma plugin install opensearch

      - name: Validate Sigma rules
        run: sigma check homelab/detection-as-code/rules/

      - name: Convert to OpenSearch/Graylog Lucene
        run: |
          sigma convert -t opensearch_lucene --without-pipeline \
            homelab/detection-as-code/rules/ssh/ \
            homelab/detection-as-code/rules/auth/ \
            homelab/detection-as-code/rules/system/ \
            | tee homelab/detection-as-code/output/graylog/converted-rules.txt

      - uses: actions/upload-artifact@v5
        with:
          name: graylog-lucene-rules
          path: homelab/detection-as-code/output/graylog/converted-rules.txt

title: New User Account Created
id: a1b2c3d4-0005-0005-0005-000000000005
status: test
description: Detects creation of a new user account on a Linux host
references:
    - https://attack.mitre.org/techniques/T1136/001/
author: Matthew
date: 2026/07/02
logsource:
    product: linux
    service: auth
detection:
    selection:
        message|contains: 'new user'
    condition: selection
falsepositives:
    - Legitimate user provisioning via Ansible or admin activity
level: high
tags:
    - attack.persistence
    - attack.t1136.001
